# Downstream Compatibility Tests

These tests check whether public cfDNAlab output schemas can be loaded by common
Python and R analysis packages. They are intentionally separate from the normal
Rust test suite because they install Python/R scientific stacks and should not be
part of the local development loop.

The current fixtures are small midpoint and end-motif Zarr V3 stores plus length-count TSV files generated by cfDNAlab itself. Python checks use the local `cfdnalab` helper package plus direct `zarr`, `xarray`, and `dask.array` readers. R checks use the local `cfdnalab` helper package plus direct CRAN `zarr` reads.

```text
midpoint counts[group, length_bin, position]
end-motif dense counts[row, motif]
end-motif sparse COO row/motif/count arrays
length-count wide TSV rows for global/windowed/grouped outputs
```

The midpoint fixture includes three groups, repeated intervals per group, three
fragment-length bins, five midpoint-position bins, varied nonzero count values
from position binning, coordinate arrays, and JSON group labels on the `group`
axis. The end-motif fixtures cover dense global output, sparse windowed output, and sparse grouped output with fixed-width ASCII motif labels. The length-count fixtures cover global, windowed, and grouped TSV outputs with broad length bins, blacklist metadata, no-blacklist windowed/grouped outputs, non-alphabetical group order, and a zero-count group. The fixtures must be generated by cfDNAlab itself. Otherwise the suite would only test reader compatibility with hand-authored approximations of the schemas.

Run locally only when you want to check downstream package compatibility:

```bash
cargo test --no-default-features --features cmd_midpoints,cmd_ends,cmd_lengths --test generate_downstream_zarr_fixtures -- --ignored
python -m pip install -r downstream_tests/requirements-python.txt
python -m pip install -e py-cfdnalab
(cd py-cfdnalab && uv run pytest)
CFDNALAB_MIDPOINT_ZARR=downstream_tests/tmp/tiny.midpoint_profiles.zarr \
CFDNALAB_ENDS_DENSE_GLOBAL_ZARR=downstream_tests/tmp/tiny_dense_global.end_motifs.zarr \
CFDNALAB_ENDS_SPARSE_WINDOWED_ZARR=downstream_tests/tmp/tiny_sparse_windowed.end_motifs.zarr \
CFDNALAB_ENDS_SPARSE_GROUPED_ZARR=downstream_tests/tmp/tiny_sparse_grouped.end_motifs.zarr \
pytest downstream_tests/python
```

R tests require the CRAN `jsonlite`, `zarr`, `qs2`, `bit64`, `Matrix`, and
`testthat` packages. The local `cfdnalab` R package must be installed before
running `test_cfdnalab_r_package.R`.

```bash
Rscript -e 'testthat::test_local("r-cfdnalab")'
R CMD INSTALL r-cfdnalab

CFDNALAB_MIDPOINT_ZARR=downstream_tests/tmp/tiny.midpoint_profiles.zarr \
CFDNALAB_ENDS_DENSE_GLOBAL_ZARR=downstream_tests/tmp/tiny_dense_global.end_motifs.zarr \
CFDNALAB_ENDS_SPARSE_WINDOWED_ZARR=downstream_tests/tmp/tiny_sparse_windowed.end_motifs.zarr \
CFDNALAB_ENDS_SPARSE_GROUPED_ZARR=downstream_tests/tmp/tiny_sparse_grouped.end_motifs.zarr \
CFDNALAB_LENGTHS_GLOBAL_TSV=downstream_tests/tmp/tiny_lengths_global.length_counts.tsv.zst \
CFDNALAB_LENGTHS_WINDOWED_TSV=downstream_tests/tmp/tiny_lengths_windowed.length_counts.tsv.zst \
CFDNALAB_LENGTHS_GROUPED_TSV=downstream_tests/tmp/tiny_lengths_grouped.length_counts.tsv.zst \
CFDNALAB_LENGTHS_WINDOWED_NO_BLACKLIST_TSV=downstream_tests/tmp/tiny_lengths_windowed_no_blacklist.length_counts.tsv.zst \
CFDNALAB_LENGTHS_GROUPED_NO_BLACKLIST_TSV=downstream_tests/tmp/tiny_lengths_grouped_no_blacklist.length_counts.tsv.zst \
Rscript downstream_tests/R/test_cfdnalab_r_package.R

CFDNALAB_MIDPOINT_ZARR=downstream_tests/tmp/tiny.midpoint_profiles.zarr \
CFDNALAB_ENDS_DENSE_GLOBAL_ZARR=downstream_tests/tmp/tiny_dense_global.end_motifs.zarr \
CFDNALAB_ENDS_SPARSE_WINDOWED_ZARR=downstream_tests/tmp/tiny_sparse_windowed.end_motifs.zarr \
CFDNALAB_ENDS_SPARSE_GROUPED_ZARR=downstream_tests/tmp/tiny_sparse_grouped.end_motifs.zarr \
Rscript downstream_tests/R/test_midpoint_zarr_cran_zarr.R

CFDNALAB_MIDPOINT_ZARR=downstream_tests/tmp/tiny.midpoint_profiles.zarr \
CFDNALAB_ENDS_DENSE_GLOBAL_ZARR=downstream_tests/tmp/tiny_dense_global.end_motifs.zarr \

```

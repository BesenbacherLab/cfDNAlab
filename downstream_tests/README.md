# Downstream Compatibility Tests

These tests check whether public cfDNAlab output schemas can be loaded by common
Python and R analysis packages and by the public Rust output loaders. They are
intentionally separate from the normal Rust test suite because they use generated
downstream fixtures and, for Python/R, install scientific stacks that should not
be part of the local development loop.

The current fixtures are small midpoint and end-motif Zarr V3 stores plus length-count TSV files generated by cfDNAlab itself. Python checks use the local `cfdnalab` helper package plus direct `zarr`, `xarray`, and `dask.array` readers. R checks use the local `cfdnalab` helper package plus direct CRAN `zarr` reads. Rust checks use `cfdnalab::output_loaders` against the same generated files.

```text
midpoint counts[group, length_bin, position]
end-motif dense counts[row, motif]
end-motif sparse COO row/motif/count arrays, including motif-group axes
length-count wide TSV rows for global/windowed/grouped outputs
```

The midpoint fixture includes three groups, repeated intervals per group, three
fragment length bins, five midpoint-position bins, varied nonzero count values
from position binning, coordinate arrays, and JSON group labels on the `group`
axis. The end-motif fixtures cover dense global output, sparse windowed output, sparse grouped output with fixed-width ASCII motif labels, ungrouped `--motifs-file` output with both inside and outside motif bases, and sparse grouped `--motifs-file` output with motif-group JSON labels. The length-count fixtures cover global, windowed, and grouped TSV outputs with broad length bins, blacklist metadata, no-blacklist windowed/grouped outputs, non-alphabetical group order, and a zero-count group. The fixtures must be generated by cfDNAlab itself. Otherwise the suite would only test reader compatibility with hand-authored approximations of the schemas.

Run locally only when you want to check downstream package compatibility:

```bash
cargo test --no-default-features --features cmd_midpoints,cmd_ends,cmd_lengths,testing --test generate_downstream_zarr_fixtures -- --ignored
cargo test --manifest-path downstream_tests/rust/Cargo.toml --no-default-features --features cmd_midpoints,cmd_ends,cmd_lengths,testing --test test_downstream_rust_loaders -- --ignored
python -m pip install -r downstream_tests/requirements-python.txt
python -m pip install -e py-cfdnalab
(cd py-cfdnalab && uv run pytest)
CFDNALAB_MIDPOINT_ZARR=downstream_tests/tmp/tiny.midpoint_profiles.zarr \
CFDNALAB_ENDS_DENSE_GLOBAL_ZARR=downstream_tests/tmp/tiny_dense_global.end_motifs.zarr \
CFDNALAB_ENDS_SPARSE_WINDOWED_ZARR=downstream_tests/tmp/tiny_sparse_windowed.end_motifs.zarr \
CFDNALAB_ENDS_SPARSE_WINDOWED_SELECTED_MOTIFS_ZARR=downstream_tests/tmp/tiny_sparse_windowed_selected_motifs.end_motifs.zarr \
CFDNALAB_ENDS_SPARSE_GROUPED_ZARR=downstream_tests/tmp/tiny_sparse_grouped.end_motifs.zarr \
CFDNALAB_ENDS_SPARSE_GROUPED_MOTIF_GROUP_ZARR=downstream_tests/tmp/tiny_sparse_grouped_motif_groups.end_motifs.zarr \
CFDNALAB_ENDS_SPARSE_GROUPED_WIDE_MOTIF_GROUP_ZARR=downstream_tests/tmp/tiny_sparse_grouped_wide_motif_groups.end_motifs.zarr \
CFDNALAB_LENGTHS_GLOBAL_TSV=downstream_tests/tmp/tiny_lengths_global.length_counts.tsv.zst \
CFDNALAB_LENGTHS_WINDOWED_TSV=downstream_tests/tmp/tiny_lengths_windowed.length_counts.tsv.zst \
CFDNALAB_LENGTHS_GROUPED_TSV=downstream_tests/tmp/tiny_lengths_grouped.length_counts.tsv.zst \
CFDNALAB_LENGTHS_WINDOWED_NO_BLACKLIST_TSV=downstream_tests/tmp/tiny_lengths_windowed_no_blacklist.length_counts.tsv.zst \
CFDNALAB_LENGTHS_GROUPED_NO_BLACKLIST_TSV=downstream_tests/tmp/tiny_lengths_grouped_no_blacklist.length_counts.tsv.zst \
pytest downstream_tests/python
```

R tests require the CRAN `jsonlite`, `zarr`, `qs2`, `bit64`, `Matrix`, and
`testthat` packages. The data.table length-count check also uses `data.table`
and the `zstd` command-line tool. The local `cfdnalab` R package must be
installed before running `test_cfdnalab_r_package.R`.

# Downstream Compatibility Tests

These tests check whether public cfDNAlab output schemas can be loaded by common
Python and R analysis packages. They are intentionally separate from the normal
Rust test suite because they install Python/R scientific stacks and should not be
part of the local development loop.

The current fixture is a small midpoint profile Zarr V3 store. Python checks
use `zarr`, `xarray`, and `dask.array`; R checks use CRAN `zarr` and
Bioconductor `Rarr`.

```text
counts[group, length_bin, position]
```

It includes three groups, repeated intervals per group, three fragment-length
bins, five midpoint-position bins, varied nonzero count values from position
binning, and the coordinate arrays and group-name fallback arrays used by the
midpoint writer. The fixture must be generated by cfDNAlab itself; otherwise the
suite would only test reader compatibility with a hand-authored approximation of
the schema.

Run locally only when you want to check downstream package compatibility:

```bash
cargo test --no-default-features --features cmd_midpoints --test generate_downstream_midpoint_fixture -- --ignored --exact generate_midpoint_zarr_fixture_with_cfdnalab
python -m pip install -r downstream_tests/requirements-python.txt
CFDNALAB_MIDPOINT_ZARR=downstream_tests/tmp/tiny.midpoint_profiles.zarr pytest downstream_tests/python
```

R tests require the CRAN `zarr` package and Bioconductor `Rarr` package.

```bash
CFDNALAB_MIDPOINT_ZARR=downstream_tests/tmp/tiny.midpoint_profiles.zarr Rscript downstream_tests/R/test_midpoint_zarr_rarr.R
CFDNALAB_MIDPOINT_ZARR=downstream_tests/tmp/tiny.midpoint_profiles.zarr Rscript downstream_tests/R/test_midpoint_zarr_cran_zarr.R
```

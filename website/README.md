# cfDNAlab docs website

This folder contains the Docusaurus website for cfDNAlab documentation.

## Local development

0. Stand in the website directory.

```bash
cd website
```

1. Install dependencies:

```bash
npm install
```

2. Start local dev server:

```bash
npm run start
```

`npm run start` generates the CLI reference, output loader reference, and release notes before starting Docusaurus.

Open the URL printed by Docusaurus in your browser.

## Build static site

```bash
npm run build
```

`npm run build` generates the CLI reference, output loader reference, and release notes before building the site.

Build output is written to `website/generated-site/`.

## CLI docs generator

The CLI reference is generated by the repository-local Rust example at `examples/gen_cli_docs.rs`, run through `website/scripts/generate_cli_docs.sh`. It is intentionally not a Cargo package binary, so `cargo install cfdnalab` installs only the public `cfdna` command.

The R/Python output loader reference is generated by `website/scripts/generate_loader_docs.py`. It reads Python docstrings from `py-cfdnalab/src/cfdnalab` and roxygen `.Rd` files from `r-cfdnalab/man`.

## Generated folders

- `website/docs/generated/cli/` is auto-generated and must not be edited manually.
- `website/docs/generated/loaders/` is auto-generated and must not be edited manually.
- `website/docs/generated/release-notes.md` is auto-generated from `CHANGELOG.md`.
- `website/generated-site/` is generated build output and must not be committed.
